    Cortex - Diagrama entidad-relación (ERD)

    Diagrama entidad-relación

    Google Cloud Cortex Data Foundation

    Google Cloud Cortex Foundation para SAP tiene dos ERD, uno para sistemas SAP ECC y otro para sistemas SAP S/4HANA.

    Conceptos

    El diagrama entidad-relación de Google Cloud Cortex Framework para el sistema SAP es una representación compleja y detallada de las relaciones entre las distintas entidades del sistema SAP. El diagrama está dividido en múltiples áreas funcionales, cada una de las cuales está representada por un color diferente. Las vistas base son la base del diagrama y están representadas por tablas azules. Las vistas de informes se construyen sobre las vistas base y están representadas por tablas verdes. Los modelos de bigquery ml se utilizan para generar información a partir de los datos en las vistas de informes y están representados por tablas amarillas.

    Las vistas base son el nivel de datos más detallado del diagrama y normalmente provienen de las tablas de SAP.

    Las vistas de informes son más agregadas y normalmente se utilizan con fines de generación de informes.

    Los modelos de bigquery ml se utilizan para generar información a partir de los datos de las vistas de informes.

    La diferencia entre una vista base y una vista de informes es que la vista base es el nivel de datos más detallado, mientras que la vista de informes es más agregada.

    La vista base se suele utilizar para el análisis de datos, mientras que la vista de informes se suele utilizar para fines de generación de informes.

    Los conceptos de ERD que se utilizan en el diagrama incluyen entidades, relaciones y atributos.

    Las entidades son los objetos que se representan en el diagrama. Las relaciones son las conexiones entre las entidades. Los atributos son las propiedades de las entidades.

    El diagrama de relaciones entre entidades de Google Cloud Cortex Framework para el sistema SAP es un recurso valioso para comprender las relaciones entre las distintas entidades del sistema SAP.

    El diagrama se puede utilizar para identificar la fuente de datos de un informe en particular, comprender las relaciones entre diferentes entidades y generar información a partir de los datos.
